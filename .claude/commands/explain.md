# Skill: Explain an AI Concept

## Usage
`/explain [concept]`

Examples:
- `/explain RAG`
- `/explain embeddings`
- `/explain tool-use`
- `/explain agent`

## Steps

1. **Identify the concept**
   - Map the input to a known AI engineering topic
   - If ambiguous, ask for clarification

2. **Explain in 3 layers**
   - **One sentence** — what it is, no jargon
   - **How it works** — the mechanism, with a simple analogy
   - **Why it matters** — what problem it solves in production systems

3. **Show it in the context of this project**
   - Find or describe where this concept appears (or will appear) in the webserver codebase
   - Reference the relevant phase in `AI/AI_Engineer_Interview_Prep.md`
   - If code already exists: show the relevant snippet and explain how the concept maps to it

4. **Interview angle**
   - What is the typical interview question that tests this concept?
   - What is the answer a mid-level AI engineer is expected to give?
   - What follow-up question should you be prepared for?

5. **Common mistakes**
   - What do people get wrong about this concept?
   - What would be a red flag answer in an interview?
