## A Strangler Fig Pattern — hogyan bontsd szét fokozatosan

Ez a bevált iparági minta: **nem dobod el a monolitot, hanem "kiszeded" belőle a funkciókat**, és egy API Gateway mögé teszed az újakat.

### 1. lépés: API Gateway bevezetése (2-3 nap tanulás)

```
UserInputServer ──► [API Gateway] ──► DatabaseServer (még monolit)
                                 └──► NotificationService (új, kicsi)
```

**Mit tanulsz:** Spring Cloud Gateway vagy Nginx mint reverse proxy, routing szabályok, load balancing alapok.

**Gyakorlat:** A meglévő Kafka `PersonCreatedEvent`-et feldolgozó NotificationService kiemelése önálló Spring Boot alkalmazásba.

---

### 2. lépés: Adatbázis szétválasztás (a legnehezebb rész!)

Az igazi mikroszerviz saját adatbázissal rendelkezik. Ezt nevezzük **Database per Service** mintának.

```
DatabaseServer ──► usersdb (PostgreSQL)
NotificationService ──► notificationsdb (PostgreSQL - külön schema)
SearchService (új) ──► searchdb (vagy Redis)
```

**Mit tanulsz:** Eventual Consistency, Saga Pattern (ha tranzakciókat kell elosztani), Flyway/Liquibase migrációk.

---

### 3. lépés: Service Discovery + Config Server

Amikor 4+ szervized van, kézzel nem menedzseled a portokat és URL-eket.

**Mit tanulsz:** Spring Cloud Eureka (service registry), Spring Cloud Config (centralizált konfiguráció).

## A K8s-ra való átállás útja

### Lokális K8s először (ingyenes!)

| Eszköz | Mire jó |
|--------|---------|
| **minikube** | Legegyszerűbb, 1 node-os K8s lokálisan |
| **kind** (K8s in Docker) | Gyorsabb, CI/CD-be is beilleszthető |
| **k3d** | k3s-t futtat Dockerben, legkönnyebb |

**Javaslat:** Kezdd `k3d`-vel — 5 perc alatt fut, és azonos a "valódi" K8s-sel.

```bash
# Így néz ki a teljes lokális stack:
Docker Desktop/Podman + k3d + Helm + kubectl
```

**Mit tanulsz K8s-ből, amit az interjúkon kérdeznek:**
- Deployment, Service, Ingress YAML-ok írása
- ConfigMap és Secret kezelés
- Liveness/Readiness probe-ok
- HorizontalPodAutoscaler
- Helm chart készítés a saját szervizedhez
- Namespace-ek és RBAC alapok

## AWS — olcsó és ingyenes lehetőségek

### 1. AWS Free Tier (12 hónap ingyenes)

| Szolgáltatás | Ingyenes limit | Mire használd |
|-------------|---------------|---------------|
| **EC2 t2.micro** | 750 óra/hó | Kis VM-ek tesztelésre |
| **EKS** | Nincs free tier! Cluster: $0.10/óra | Csak ha muszáj |
| **RDS t2.micro** | 750 óra/hó | Managed PostgreSQL |
| **ECR** | 500 MB/hó | Docker image tárolás |
| **ECS + Fargate** | Nincs free tier | Drágább mint EKS |

**Fontos:** Az EKS drága (kb. $70/hó a control plane + worker node-ok). **Ne indítsd el, ha csak tanulsz!**

### 2. LocalStack — ingyenes AWS szimulátor (legjobb tanuláshoz!)

```bash
docker run -d localstack/localstack
```

A LocalStack szimulálja az AWS szolgáltatásokat lokálisan: S3, SQS, SNS, DynamoDB, Lambda — **teljesen ingyenes**, és a Terraform kódod ugyanúgy fut rajta.

**Ez a legjobb tanulási mód:** fejlesztesz LocalStack-en, és ha kész, ugyanaz a Terraform kód megy az éles AWS-re.

### 3. Olcsó K8s alternatívák AWS helyett

| Platform | Ár | Előny |
|---------|-----|-------|
| **Linode/Akamai LKE** | $12/hó 1 node | Legolcsóbb managed K8s |
| **DigitalOcean DOKS** | $12/hó | Jó doksi, egyszerű |
| **Civo** | $5/hó | K3s alapú, nagyon olcsó |
| **Google GKE Autopilot** | Pay-per-pod | Csak használt erőforrás |
| **AWS EKS** | $70+/hó | Drágább, de CV-be kerül |

**Tanácsom:** Tanulj lokálisan k3d-vel, majd ha CV-be kell az AWS tapasztalat, indítsd el az EKS-t **1-2 napra** (kb. $2-3 cost), csináld meg amit kell, majd töröld le.

## Konkrét, következő lépések sorrendben

```
1. [Most]     k3d telepítés + a meglévő docker-compose.yml átírása K8s YAML-ra
2. [1. hét]   NotificationService kiemelése (Kafka consumer → önálló service)
3. [2. hét]   Helm chart a DatabaseServer-hez és NotificationService-hez
4. [3. hét]   API Gateway (Spring Cloud Gateway) bevezetése
5. [4. hét]   Prometheus + Grafana K8s-ben (Helm-mel telepítve)
6. [5-6. hét] LocalStack + Terraform: S3 bucket, SQS queue szimulálva
7. [7-8. hét] GitHub Actions: build → Docker image push → k3d deploy
8. [Végén]    1-2 napra éles EKS deploy, screenshot, törlés — CV-nek elég!
```

## Összefoglalás

**A legjobb stratégia:** Strangler Fig Pattern + lokális k3d + LocalStack kombinációja. Így **nullás költségen** megtanulod az összes szükséges technológiát (K8s, Helm, AWS IaC), és csak a CV-hez szükséges éles AWS deploy kerül pénzbe — azt is minimalizálhatod.

A fejlesztési tervedben a 4. és 5. fázis pontosan erre mutat, szóval a terv jó — csak a K8s lokális bevezetése hiányzik belőle explicit lépésként.
